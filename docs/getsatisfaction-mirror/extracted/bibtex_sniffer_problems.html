BibTex Sniffer problems

Recently, Qiqqa seems much less successful at automatially finding BibTex information for new additions. The following sequence happens occasionally, but not always (using v.75s). I have not seen this sequence reported here before.

After opening my library, I click on "Add new PDF documents" (top left) to import a new PDF file into Qiqqa, and select a single new PDF file.

The new file appears as the bottom citation in my library; I click on this and it opens in the main screen. To the right, the Preview tab is selected; I click on the Properties tab, which says Qiqqa was unable to find BibTex information for this article. So I add some partial information manually to the pink fields under Reference Details, and then click on Find.

A dialog box appears: "Qiqqa was unable to automatically find BibTex..." I click Yes to run the BibTex Sniffer.

Two dialog boxes appear, in quick succession. First, "There are no PDFs in this library without BibTex..." This is covered by a second dialog box, with a red X: "Please first select a PDF in your library..." I click on OK to make the top dialog box go away, and then OK for the one it covered. [I had not seen this red X box before.]

Qiqqa's BibTex & PubMed Sniffer appears (sometimes on top; sometimes hidden under the main screen). The top part shows the top of the title page for my newest article. The lower part, with "Browsing" selected, shows some HTML code (i.e., @article (Turner1913literature,
title=(title), author=Turner, CH), journal=(Journal of Animal Behavior), -- etc., through publisher. [This information is correct for my selected article.]

I manually click on the "Google Scholar" tab (to the right of "Browsing") [previously, Qiqqa automatically went to Google Scholar]. Now I see the correct citation for this paper in the lower screen, with its title as a link; I click on this link. I see a site showing the correct article (in this case, APA's PsycNET). I click on the green thumbs up icon, in top right quadrant of the Sniffer dialog box.

I see "There are no more PDFs in this library without BibTex..." I click OK.

Below the Properties tab for the open article, the Reference Details fields remain pink (they don't turn gray as when Qiqqa adds information), and nothing has been added beyond what I entered manually.

I hope this information is useful and that something can be done to fix the recent problems with Qiqqa's sniffer...

    Thanks for your response.

    I am now using v76s, and used it to add another eight or so PDF files to my library (most scanned from texts at the library). Sniffer opens with Gogle Scholar selected, now. That's good.

    For each of my most recent additions (using v.76s), Qiqqa initially could not find BibTex information; Google Scholar found correct information so I added it manually.

    But I still see code displayed. [Its now in the top right quadrant of the upper Sniffer pane; in v75s it was lower and to the left.] This code overlays the thumbs up / thumbs down icons to manually add or reject selected BibTex for a paper.

    Below, is the code for a sample recently-added reference (this PDF was available online from Springer):

    @article{Cuadras-1981
    , author = {Cuadras-1981}
    , title = {Behavioral determinants of severe cardiac inhibition}
    , journal = {Physiological Psychology}
    , year = {1981}
    , volume = {9}
    , number = {4}
    , pages = {384-392}
    , abstract = {Behaviorally elicited cardiac arrests have been observed in different phyla independently of the neurogenicity or myogenicity of pacemakers. Startling stimuli such as visual detection of close and fast movement is especially associated with complete cardiac inhibition. In contrast, orienting responses are associated with bradycardia. Although studies on vertebrates are not usually concerned with reflex responses, it can be assumed that actual cardiac arrests occur in some cases in which bradycardia is reported, but that these arrests cannot be recorded because of the methodology used. Heart rate can be used to separate patterns displayed in such defensive reflex behaviors as orientation, startle, and freezing. Behavioral sequences can be categorized into various series of patterns which correspond to various sequential and multiphasic heart-rate patterns. }

    I'm thinking this code should not be visible and should not obscure those icons, but I'm not seeing anyone else complain about seeing this. Is my installation of Qiqqa (using Win 7) corrupted somehow, or what?

    I do appreciate your help with this.

    David, that 'code' is the BibTeX record that Qiqqa has associated with your PDF.  It is in code form so that you can check its details are correct before moving onto the next PDF.  It is the same 'code' that underlies the BibTeX Grid screen - except it has then been rendered into the grid.
